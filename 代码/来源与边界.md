# 代码来源与提交边界

本仓库提交的是本次实训在虚拟机 `~/unitree_h1_learn` 中实际使用的**实验工程代码快照**，包括 H1 数据采集、MuJoCo/IK、ACT 训练与推理、验证、训练监控，以及为本次实验证据编写的适配脚本。完整快照在 `vm_source/`，并提供同内容的 ZIP 与 TAR.GZ。

## 我们提交的工程代码

- `DataCollecter/`、`Mujoco_env/`、`ACT/`：H1 仿真、采集、训练和推理的工程代码及必要 XML；
- `validation/record_h1_red_stick_expert_video.py`、`validation/validate_h1_hdf5.py`：H1 数据与视频验证；
- `validation/record_robojudo_g1_loco_dance_switch.py`、`validation/run_robojudo_beyondmimic_evidence.py`：为行走—舞蹈策略切换编写的离屏录制和有界验证适配脚本；
- `ACT/config.cpu.yaml`、`DataCollecter/analyze_hdf5.py`、`project_execution/scripts/`：本次 CPU 训练、数据检查及训练监控所用代码。

上述快照可能包含实验工程既有的基础组件；本仓库不将所有文件表述为从零原创。文件内已有的作者、版权或许可证声明应保留并遵守。

## 外部仓库：RoboJuDo

**[HansZ8/RoboJuDo](https://github.com/HansZ8/RoboJuDo)** 是本次 G1 行走/舞蹈策略管线的外部依赖。为使学弟可在克隆本仓库后离线阅读、安装和研究，本仓库在 `third_party/RoboJuDo/` 中保留了其 `release` 分支固定提交 `ed7601f` 的完整工作树快照。

- 该第三方代码与我们的工程代码分目录存放，保留其 `LICENSE`；
- `THIRD_PARTY_NOTICES.md` 与 `third_party/RoboJuDo.UPSTREAM.md` 记录上游、作者、CC BY 4.0 许可证、提交号和快照完整性；
- `validation/record_robojudo_g1_loco_dance_switch.py` 是本工程的验证适配脚本，它调用已安装的 RoboJuDo 配置/键位接口来产生视频和日志；
- 上游可选子模块没有复制；如需真实机器人 SDK 或附加模块，请在 `third_party/RoboJuDo/` 内按其 README 和 `submodule_cfg.yaml` 运行安装步骤。

GR00T、Unitree SDK、原始 HDF5 数据及训练检查点仍未提交：它们或为外部依赖，或体积过大且不适合纳入课程提交仓库。
